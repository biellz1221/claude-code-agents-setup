# Grand Slam Offer — Framework Alex Hormozi

## Core Philosophy

"Make people an offer so good they feel stupid saying no."

The problem is never the product. The problem is the OFFER.
A Grand Slam Offer makes the value SO obvious and the risk SO low
that buying becomes the logical choice.

---

## The Value Equation

```
                Dream Outcome × Perceived Likelihood of Achievement
Value = ──────────────────────────────────────────────────────────────
              Time Delay × Effort & Sacrifice

To INCREASE value:
  ↑ Dream Outcome         → Make the end result bigger/better
  ↑ Perceived Likelihood  → Add proof, guarantees, case studies
  ↓ Time Delay            → Faster results, instant access
  ↓ Effort & Sacrifice    → Make it easier, done-for-you elements
```

**In practice for SaaS:**
- Dream Outcome: "Save 10 hours per week" (specific, measurable)
- Likelihood: "2,000+ businesses already use it" (social proof)
- Time Delay: "See results in your first session" (fast time-to-value)
- Effort: "Setup takes 5 minutes" (low effort)

## Step 1: Identify the Dream Outcome

Ask: "If my customer could wave a magic wand, what would their ideal result be?"

**Framework for SaaS dream outcomes:**
```
Category          | Dream Outcome Template
──────────────────|──────────────────────────────────────
Save Time         | "Get back [X] hours per [week/month]"
Save Money        | "Reduce [expense] by [X]% "
Make Money        | "Increase [revenue/leads/conversions] by [X]%"
Reduce Risk       | "Never worry about [bad thing] again"
Gain Status       | "Look like a pro at [thing]"
Reduce Pain       | "Eliminate [frustrating task] forever"
```

**Rules:**
- Be SPECIFIC (not "save time" but "save 10 hours per week on manual reporting")
- Use their language (not your feature names)
- Quantify when possible (numbers > vague promises)

---

## Step 2: List All Problems (The Obstacle Map)

Before solving, list EVERY problem between the customer and the dream outcome.

**Problem categories:**
```
BEFORE Purchase:
  "Will this actually work for me?"
  "Can I afford it?"
  "What if I don't have time to set it up?"
  "What if my team won't use it?"

DURING Use:
  "How do I set it up?"
  "What if I get stuck?"
  "How do I migrate my data?"
  "How long until I see results?"

AFTER Purchase:
  "What if it breaks?"
  "What if I need help?"
  "What if my needs change?"
  "What if something better comes along?"
```

**For each problem, create a SOLUTION.**
These solutions become the components of your offer.

## Step 3: Build the Offer Stack

The offer is NOT just the product. It's the product + everything that makes success inevitable.

### The Stack Structure

```markdown
## The Grand Slam Offer for [Product]

### Core Product — Value: R$[X]/month
[The main product with all features]

### Bonus 1: [Name] — Value: R$[X]
[Solves problem: setup/onboarding]
Example: "Quick-Start Setup Guide — templates pré-configurados
  pra você estar rodando em 15 minutos, não em 3 dias"

### Bonus 2: [Name] — Value: R$[X]
[Solves problem: knowledge gap]
Example: "Masterclass: [Topic] — 5 vídeos de 10 min que te ensinam
  exatamente como [get specific result] usando [Product]"

### Bonus 3: [Name] — Value: R$[X]
[Solves problem: ongoing support]
Example: "Comunidade VIP — acesso ao grupo exclusivo de [niche]
  onde você tira dúvidas direto comigo"

### Bonus 4: [Name] — Value: R$[X]
[Solves problem: time/effort]
Example: "Template Pack — [N] templates prontos pra usar.
  Economiza [X] horas de trabalho manual"

### Bonus 5: [Name] — Value: R$[X]
[Solves problem: risk/uncertainty]
Example: "Setup Done-For-You — eu configuro tudo pra você
  nos primeiros 30 minutos (para os primeiros 50 clientes)"

### TOTAL VALUE: R$[sum]

### YOUR PRICE: R$[fraction of total value]

### GUARANTEE: [see step 5]
```

### Bonus Naming Rules
- Name each bonus like a standalone product
- Assign a monetary value to each (anchor the total value)
- Each bonus must solve a SPECIFIC problem from the obstacle map
- Each bonus should make the core product MORE likely to work
- At least one bonus should be "fast action" (urgency/scarcity)

## Step 4: Apply the Pricing Framework

### Price Anchoring
```
Show the VALUE first, THEN reveal the price.

"O valor total de tudo isso é R$2.970.
Mas você não vai pagar nem perto disso.
Seu investimento hoje é de apenas R$97/mês.
Isso dá R$3,23 por dia.
Menos do que um café."
```

### Pricing Strategies

**1. The "No-Brainer" Price**
```
If your product saves 10h/week and the customer's time is worth R$50/h:
Value delivered: R$2000/month
Your price: R$97/month (5% of value)
ROI: 20x — it's stupid NOT to buy
```

**2. The "Compare to Alternative" Price**
```
"Contratar alguém pra fazer isso custa R$3.000/mês.
[Product] faz o mesmo por R$97/mês.
Você economiza R$2.903/mês."
```

**3. The "Cost of NOT Buying" Price**
```
"Cada dia sem [Product], você perde R$X em [time/money/leads].
Em um mês, isso são R$[X×30].
O investimento de R$97 se paga em [X] dias."
```

## Step 5: Craft the Guarantee

**The guarantee's job: eliminate ALL risk from the buyer's side.**

### Guarantee Types (strongest → weakest)

**1. Unconditional Money-Back**
```
"Teste por 30 dias. Se não gostar por qualquer motivo, devolvemos 100%.
Sem perguntas, sem burocracia. Resposta em 24h."
```

**2. Conditional Results-Based**
```
"Use [Product] por 60 dias seguindo o método. Se não [specific result],
devolvemos 100% + te pagamos R$100 pelo seu tempo."
```

**3. Double-Your-Money-Back (nuclear option)**
```
"Se em 90 dias você não tiver [specific result],
devolvemos o DOBRO do que pagou. Sério."
```

**4. Keep-Everything Guarantee**
```
"Se pedir reembolso, pode manter todos os bônus.
Sem ressentimentos."
```

**Which to choose:**
- New product, no social proof: STRONGEST guarantee (reduce risk to zero)
- Established product with proof: Moderate guarantee + heavy social proof
- The guarantee should match the dream outcome claim

---

## Step 6: Add Urgency & Scarcity (REAL only)

### Legitimate Scarcity for SaaS
```
✅ "Preço de lançamento: R$47/mês. Após [date], sobe pra R$97/mês."
   (Actually change the price on that date)

✅ "Bônus de setup personalizado: apenas para os primeiros 50 clientes."
   (You actually can only onboard 50 people personally)

✅ "Comunidade VIP: limitada a 200 membros pra manter qualidade."
   (Genuine capacity constraint)

✅ "Desconto de early adopter: 40% OFF vitalício."
   (Rewarding early customers is fair)

❌ "ÚLTIMAS VAGAS!!!" (for a digital product with infinite capacity)
❌ Fake countdown timers that reset
❌ "Oferta termina hoje!" (every day, forever)
```

## Process

1. **Understand the product** — read codebase, features, current pricing
2. **Define the dream outcome** — the #1 result customers want
3. **Map all obstacles** — every reason someone wouldn't buy or wouldn't succeed
4. **Create solutions for each obstacle** → these become bonuses
5. **Stack the offer** — core product + bonuses + value anchoring
6. **Set the price** — using the value equation and comparison frameworks
7. **Craft the guarantee** — eliminate all buyer risk
8. **Add real urgency** — legitimate scarcity only
9. **Write the offer page** — complete offer stack ready to use

Generate a complete Grand Slam Offer document with:
- Value equation breakdown
- Obstacle map
- Offer stack with pricing
- Guarantee
- Urgency mechanism
- Complete offer copy ready for landing page or sales page
